## üì∞ Title Optimization

### Best Practices
‚úÖ **DO:**
- Keep between 40-60 characters (including spaces)
- Include primary keyword near the beginning
- Make it compelling and click-worthy
- Use numbers when relevant (e.g., "5 Ways to...")
- Be specific and descriptive
- Use title case capitalization

‚ùå **DON'T:**
- Exceed 70 characters (gets cut off in search results)
- Use clickbait or misleading titles
- Stuff with too many keywords
- Use all caps or excessive punctuation
- Be vague or generic

### Good Examples
- ‚úÖ "Building a Real-Time Object Detection System with YOLOv8"
- ‚úÖ "5 Advanced PyTorch Techniques for Faster Training"
- ‚úÖ "How I Reduced Model Inference Time by 70%"
- ‚úÖ "Deep Learning for Time Series: A Practical Guide"

### Poor Examples
- ‚ùå "My Project" (too vague)
- ‚ùå "AMAZING BREAKTHROUGH IN AI THAT WILL SHOCK YOU!!!" (clickbait)
- ‚ùå "Machine Learning Deep Learning AI Neural Networks Tutorial Guide" (keyword stuffing)
- ‚ùå "This is an extremely long title that goes on and on and gets cut off in search results which is bad" (too long)

## üìù Description/Excerpt Guidelines

### Best Practices
‚úÖ **DO:**
- Write 140-160 characters (sweet spot for search results)
- Include primary and secondary keywords naturally
- Make it a compelling summary
- Include a benefit or outcome
- End with a call to action if appropriate
- Write unique descriptions for each article

‚ùå **DON'T:**
- Copy the first sentence of the article
- Exceed 160 characters (gets truncated)
- Be under 120 characters (wasted opportunity)
- Stuff with keywords unnaturally
- Use generic descriptions

### Good Examples
- ‚úÖ "Learn how to implement YOLOv8 for real-time object detection. Complete code walkthrough, optimization tips, and performance benchmarks included." (149 chars)
- ‚úÖ "Discover 5 PyTorch techniques that cut my model training time in half. Practical examples with code snippets and implementation details." (136 chars)
- ‚úÖ "A practical guide to applying deep learning for time series forecasting. Covers LSTMs, Transformers, and best practices." (122 chars)

### Poor Examples
- ‚ùå "This article is about machine learning." (too short, 41 chars)
- ‚ùå "In this article, I'm going to talk about..." (boring opening)
- ‚ùå "Machine learning deep learning AI neural networks PyTorch TensorFlow optimization training inference deployment production..." (keyword stuffing, 128 chars)

## üîó Slug (URL) Best Practices

### Guidelines
- Use lowercase only
- Separate words with hyphens (-)
- Include primary keyword
- Keep it concise (3-5 words ideal)
- Remove stop words when possible (the, a, an, of, etc.)
- Match the article title (simplified)

### Good Examples
- ‚úÖ `realtime-object-detection-yolov8`
- ‚úÖ `pytorch-training-optimization-tips`
- ‚úÖ `reduce-model-inference-time`
- ‚úÖ `deep-learning-time-series-guide`

### Poor Examples
- ‚ùå `My_Article_123` (inconsistent, unclear)
- ‚ùå `building-a-real-time-object-detection-system-with-yolov8-complete-guide` (too long)
- ‚ùå `article-2024-11-02` (no keywords)
- ‚ùå `ML Tutorial!` (uppercase, special chars, space)

## ‚úçÔ∏è Article Content Best Practices

### Structure
1. **Introduction** (1-2 paragraphs)
   - Hook the reader
   - State what they'll learn
   - Include primary keyword

2. **Body** (Main content)
   - Use H2 and H3 headings
   - Break into scannable sections
   - Include code examples
   - Add images/diagrams
   - Use bullet points and lists

3. **Conclusion** (1-2 paragraphs)
   - Summarize key points
   - Include call to action
   - Link to related articles

### Heading Hierarchy
```
# Article Title (H1 - automatically from title field)

## Main Section (H2)
Introduction to the topic...

### Subsection (H3)
Detailed explanation...

## Another Main Section (H2)
More content...

### Another Subsection (H3)
Additional details...
```

### Content Length
- **Minimum**: 800-1000 words
- **Optimal**: 1500-2500 words
- **In-depth guides**: 3000+ words
- **Quality over quantity**: Better to have 1000 words of great content than 3000 words of fluff

### Keyword Usage
- Include primary keyword in:
  - Title
  - First paragraph
  - At least one H2 heading
  - URL/slug
  - Meta description
  - Naturally throughout content

- **Keyword Density**: 1-2% (naturally, not forced)
- **LSI Keywords**: Use related terms and synonyms
- **Avoid Keyword Stuffing**: Write for humans first, search engines second

- Use syntax highlighting

## üîç Keyword Research Tips

### Find Keywords
1. **Google Autocomplete**: Start typing in Google, see suggestions
2. **People Also Ask**: Check related questions in search results
3. **Related Searches**: Bottom of Google search results page
4. **Competitors**: See what similar portfolios rank for
5. **Google Trends**: Check interest over time

### Evaluate Keywords
- **Search Volume**: Higher is better (but competitive)
- **Difficulty**: Target low-medium difficulty
- **Intent**: Match what people want to find
- **Relevance**: Must relate to your expertise

### Long-Tail Keywords
- More specific phrases (3-5 words)
- Lower competition
- Higher conversion
- Example: "pytorch lstm time series forecasting" vs "machine learning"
